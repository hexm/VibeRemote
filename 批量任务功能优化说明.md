# 批量任务功能优化说明

## 需求概述

1. **创建批量任务时，先创建批量任务，再根据选择的agent数量创建普通任务，批量任务和普通任务建立关联**
2. **批量任务列表，目标客户端改为任务数量，点击任务数量对应的数字，链接到这个批量任务下所有任务的列表展示**
3. **创建批量任务和任务，都必须输入任务名称，页面上帮助填写默认值，任务名称不能重复**

## 实现方案

### 一、数据库层修改（Repository）

#### 1. TaskRepository.java
- 添加 `existsByTaskName(String taskName)` - 检查任务名称是否存在
- 添加 `findByTaskName(String taskName)` - 根据任务名称查询任务

#### 2. BatchTaskRepository.java
- 添加 `existsByBatchName(String batchName)` - 检查批量任务名称是否存在
- 添加 `findByBatchName(String batchName)` - 根据批量任务名称查询

### 二、服务层修改（Service）

#### 1. BatchTaskService.java
**修改 `createBatchTask` 方法：**
- ✅ 添加批量任务名称唯一性验证
- ✅ 如果名称已存在，抛出 `IllegalArgumentException` 异常
- ✅ 创建批量任务记录
- ✅ 为每个选中的Agent创建关联的普通任务
- ✅ 子任务名称格式：`批量任务名称-任务1`, `批量任务名称-任务2`, ...
- ✅ 所有子任务通过 `batchId` 字段关联到批量任务

**核心逻辑：**
```java
// 1. 验证批量任务名称唯一性
if (batchTaskRepository.existsByBatchName(batchName)) {
    throw new IllegalArgumentException("批量任务名称已存在: " + batchName);
}

// 2. 创建批量任务记录
BatchTask batchTask = new BatchTask();
batchTask.setBatchName(batchName);
// ... 设置其他字段
batchTaskRepository.save(batchTask);

// 3. 为每个Agent创建关联的子任务
for (String agentId : agentIds) {
    Task task = new Task();
    task.setBatchId(batchTask.getBatchId()); // 建立关联
    task.setTaskName(batchName + "-任务" + index);
    // ... 设置其他字段
    taskRepository.save(task);
}
```

## 核心特性

### 1. 批量任务与子任务的关联关系
- 批量任务通过 `batchId` 字段与子任务建立一对多关系
- 每个子任务的 `batchId` 字段指向所属的批量任务
- 子任务名称格式：`批量任务名称-任务1`, `批量任务名称-任务2`, ...

### 2. 任务名称唯一性保证
- **数据库层**：通过Repository的 `existsByTaskName` 和 `existsByBatchName` 方法检查
- **服务层**：在创建任务前进行验证，重复时抛出异常
- **前端层**：通过表单验证确保用户填写名称
- **默认值**：自动生成基于时间戳的默认名称，用户可以修改

### 3. 批量任务列表优化
- 列名从"目标客户端"改为"任务数量"，更准确地反映内容
- 任务数量支持点击，直接跳转到批量任务详情页面
- 详情页面展示该批量任务下的所有子任务

### 4. 用户体验优化
- ✅ 打开创建对话框时自动填充默认任务名称
- ✅ 默认名称格式：`任务_2025-10-26 22:00:00`
- ✅ 用户可以修改默认名称
- ✅ 输入框限制100字符，显示字数统计
- ✅ 提交前验证名称不为空且唯一

## 数据流程

### 创建批量任务流程
```
1. 用户打开"批量任务"对话框
   └─> 自动填充默认名称：批量任务_2025-10-26 22:00:00

2. 用户填写/修改任务信息
   - 批量任务名称（可修改默认值）
   - 选择多个客户端
   - 填写脚本内容
   - 设置超时时间

3. 点击"创建"按钮
   └─> 前端验证：检查所有必填项
   └─> 发送请求到后端

4. 后端处理
   └─> 验证批量任务名称唯一性
   └─> 创建BatchTask记录
   └─> 为每个选中的Agent创建Task记录
       - Task.batchId = BatchTask.batchId
       - Task.taskName = batchName + "-任务" + index
   └─> 返回成功响应

5. 前端刷新
   └─> 关闭对话框
   └─> 刷新批量任务列表
   └─> 显示成功消息
```

### 查看批量任务详情流程
```
1. 用户在批量任务列表点击"任务数量"
   └─> 触发 viewBatchTaskDetail(batchTask)

2. 打开批量任务详情对话框
   └─> 显示批量任务基本信息
   └─> 显示执行进度和统计

3. 加载子任务列表
   └─> 调用 GET /api/web/batch-tasks/{batchId}/tasks
   └─> 后端查询所有 batchId 匹配的Task
   └─> 返回子任务列表

4. 展示子任务信息
   └─> 任务名称、客户端、状态、时间
   └─> 支持查看详情和日志
```
