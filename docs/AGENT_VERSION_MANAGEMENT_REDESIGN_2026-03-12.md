# Agent版本管理重新设计：文件上传驱动

## 文档信息
- **创建时间**: 2026-03-12
- **版本**: 1.0
- **状态**: 已实施
- **作者**: 系统架构师

## 设计背景

用户提出了更简洁的版本管理设计思路：
1. **版本只能通过文件上传**，上传后版本保存到服务器上
2. **自动通过文件名识别版本号**，保留文件名、文件大小、哈希、版本说明
3. **管理员上传时只需要填写版本说明**
4. **强制升级选择放在列表中**，可以直接在列表中修改（开关）

## 新设计方案

### 核心改进

#### 1. 文件上传驱动的版本创建
- **必须上传文件**：版本记录必须基于实际的JAR文件创建
- **文件存储**：利用现有的文件管理系统存储版本文件
- **关联管理**：版本记录与文件记录关联，确保一致性

#### 2. 自动版本号解析
```java
// 从文件名解析版本号
private String parseVersionFromFilename(String filename) {
    // 支持格式：agent-1.2.3.jar, lightscript-agent-2.0.0.jar 等
    Pattern pattern = Pattern.compile("(\\d+\\.\\d+\\.\\d+(?:\\.\\d+)?)");
    Matcher matcher = pattern.matcher(filename);
    return matcher.find() ? matcher.group(1) : null;
}
```

#### 3. 简化的用户界面
- **上传表单**：只需要选择文件和填写版本说明
- **自动填充**：文件名、大小、哈希值自动获取
- **列表操作**：强制升级开关直接在表格中操作

#### 4. 基于版本号的自动管理
- **最新版本**：自动根据版本号判断，无需手动标记
- **升级逻辑**：Agent自动升级到版本号最高的版本
- **版本比较**：支持语义化版本号比较

### 工作流程

#### 1. 版本创建流程
```
1. 管理员选择JAR文件（如：agent-1.2.3.jar）
2. 填写版本说明
3. 点击"上传并创建版本"
4. 系统上传文件到文件管理系统
5. 从文件名解析版本号（1.2.3）
6. 获取文件信息（大小、哈希值）
7. 创建版本记录
8. 设置下载URL为文件系统路径
```

#### 2. 版本管理流程
```
1. 版本列表显示所有已上传的版本
2. 自动标记版本号最高的为"最新"
3. 管理员可直接在列表中切换强制升级开关
4. Agent心跳时自动检测最新版本并升级
```

#### 3. 文件命名规范
支持的文件名格式：
- `agent-1.2.3.jar`
- `lightscript-agent-2.0.0.jar`
- `my-agent-1.0.0.jar`

版本号解析规则：
- 匹配模式：`\d+\.\d+\.\d+(?:\.\d+)?`
- 支持三位或四位版本号：`1.2.3` 或 `1.2.3.4`

## 结论

### 核心改进
1. **文件驱动**：版本创建必须基于实际文件上传
2. **自动解析**：从文件名自动识别版本号和文件信息
3. **简化操作**：管理员只需要上传文件和填写说明
4. **列表管理**：强制升级等设置直接在列表中操作

### 用户价值
- **操作简化**：上传文件即可创建版本，无需手动填写复杂信息
- **错误减少**：自动解析避免手动输入错误
- **管理高效**：列表内直接操作，无需打开编辑弹窗

### 技术价值
- **数据一致性**：版本记录与实际文件强关联
- **系统可靠性**：基于文件系统的版本管理更可靠

这个重新设计大大简化了版本管理流程，提高了系统的可用性和可靠性。
