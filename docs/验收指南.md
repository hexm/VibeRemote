# LightScript 多目标任务功能验收指南

## 访问地址
- 前端界面: http://8.138.114.34:3000
- 后端API: http://8.138.114.34:8080
- 默认账号: admin / admin123

## 验收要点

### 1. 创建多代理任务
1. 登录系统
2. 进入"任务管理"页面
3. 点击"创建任务"
4. 选择多个代理（如: agent-1, agent-2, agent-3）
5. 填写任务信息
6. 提交创建

**预期结果**: 
- 任务创建成功
- 显示"已分配给 3 个代理"
- 可以看到任务ID

### 2. 查看任务执行实例
1. 在任务列表中找到刚创建的任务
2. 点击查看详情
3. 查看"执行实例"标签页

**预期结果**:
- 显示3个执行实例
- 每个实例对应一个代理
- 每个实例有独立的状态和执行次数

### 3. 查看任务摘要
1. 在任务详情页查看摘要信息

**预期结果**:
- 显示聚合状态（如: IN_PROGRESS, ALL_SUCCESS）
- 显示目标代理数: 3
- 显示执行进度: X/3
- 显示各状态统计（成功数、失败数、运行中、待处理）

### 4. Agent执行任务流程
可以通过以下API测试Agent执行流程：

```bash
# 1. 注册Agent
curl -X POST "http://8.138.114.34:8080/api/agent/register" \
  -H "Content-Type: application/json" \
  -d '{"agentId":"test-agent","hostname":"test","osType":"Linux","osVersion":"Ubuntu","registerToken":"dev-register-token"}'

# 2. 拉取任务
curl "http://8.138.114.34:8080/api/agent/tasks/pull?agentId={agentId}&agentToken={token}&max=5"

# 3. 确认任务
curl -X POST "http://8.138.114.34:8080/api/agent/tasks/executions/{executionId}/ack?agentId={agentId}&agentToken={token}"

# 4. 提交日志
curl -X POST "http://8.138.114.34:8080/api/agent/tasks/executions/{executionId}/log" \
  -H "Content-Type: application/json" \
  -d '{"agentId":"{agentId}","agentToken":"{token}","executionId":{id},"seq":1,"stream":"stdout","data":"log content\n"}'

# 5. 完成任务
curl -X POST "http://8.138.114.34:8080/api/agent/tasks/executions/{executionId}/finish" \
  -H "Content-Type: application/json" \
  -d '{"agentId":"{agentId}","agentToken":"{token}","executionId":{id},"exitCode":0,"status":"SUCCESS","summary":"OK"}'
```

### 5. 验证任务重启功能
1. 找到一个已完成的任务
2. 点击"重启任务"按钮
3. 查看执行实例列表

**预期结果**:
- 为每个代理创建新的执行实例
- 执行次数递增（如: 从1变为2）
- 新实例状态为PENDING

## 核心功能清单

### ✅ 已实现功能
1. 多代理任务创建 - 一个任务可分配给多个代理
2. 执行实例管理 - 每个代理有独立的执行实例
3. 执行次数跟踪 - 支持任务重启，执行次数自动递增
4. 状态聚合 - 自动计算任务的整体状态
5. Agent API重构 - 使用executionId替代taskId
6. 日志文件管理 - 每个执行实例独立的日志文件
7. 任务重启 - 支持重新执行任务
8. 执行实例取消 - 支持取消单个执行实例

### 数据库变更
- 新增 task_execution 表
- Task表移除 status, agent_id, batch_id 字段
- 所有状态管理迁移到 task_execution 表

### API变更
- Agent API全部使用 executionId
- Web API新增执行实例相关接口
- 任务摘要API返回聚合状态
