## 实施状态

#### 1. **Agent端组件**
- ✅ `TaskStatusMonitor` - 任务状态监控器
  - 跟踪正在执行的任务
  - 防止升级时中断正在执行的任务
  - 提供任务计数和状态查询

- ✅ `UpgradeStatusReporter` - 升级状态报告器
  - 向服务端报告升级开始
  - 报告升级过程中的状态变化
  - 处理升级完成和失败状态

- ✅ `UpgradeExecutor` - 升级执行器
  - 处理版本检查响应
  - 下载新版本文件
  - 启动独立升级器
  - 调度升级重试机制

#### 2. **增强现有组件**
- ✅ `AgentApi` - 增强心跳处理
  - 心跳响应返回版本检查信息
  - 处理服务端版本更新通知

- ✅ `SimpleTaskRunner` - 集成任务监控
  - 任务开始时注册到监控器
  - 任务完成时从监控器注销
  - 支持升级期间的任务保护

- ✅ `AgentMain` - 集成升级功能
  - 创建升级相关组件
  - 处理心跳响应中的版本检查
  - 支持升级重试调度

#### 3. **独立升级器程序**
- ✅ `upgrader.jar` - 独立升级程序
  - 跨平台兼容（Windows/Linux/macOS）
  - 完整的备份和回滚机制
  - 进程管理和验证
  - 状态报告功能

## 升级场景测试

### 场景1: 正常升级（无任务执行）

**前置条件**:
- Agent正常运行，版本1.0.0
- 服务端有新版本1.1.0可用
- 当前无任务执行

**测试步骤**:
1. Agent发送心跳到服务端
2. 服务端返回版本检查信息（有更新）
3. Agent检查任务状态：无正在执行的任务
4. Agent报告升级开始
5. Agent下载新版本文件
6. Agent启动升级器程序
7. Agent主进程退出
8. 升级器执行升级流程
9. 升级器启动新版本Agent
10. 新版本Agent开始运行

**预期结果**:
- 升级成功完成
- 新版本Agent正常运行
- 升级日志记录完整
- 无数据丢失

### 场景2: 有任务执行时的升级

**前置条件**:
- Agent正常运行，版本1.0.0
- 服务端有新版本1.1.0可用
- 当前有任务正在执行

**测试步骤**:
1. Agent发送心跳到服务端
2. 服务端返回版本检查信息（有更新）
3. Agent检查任务状态：有正在执行的任务
4. Agent调度升级重试（30分钟后）
5. 30分钟后重新检查任务状态
6. 如果任务已完成，开始升级流程
7. 如果任务仍在执行，继续等待

**预期结果**:
- 升级被延迟到任务完成后
- 正在执行的任务不被中断
- 升级最终成功完成

### 场景3: 强制升级

**前置条件**:
- Agent正常运行，版本1.0.0
- 服务端有强制升级版本1.1.0
- 当前有任务正在执行

**测试步骤**:
1. Agent发送心跳到服务端
2. 服务端返回强制升级信息
3. Agent检查任务状态：有正在执行的任务
4. Agent调度升级重试（5分钟后，因为是强制升级）
5. 5分钟后开始升级流程（即使有任务执行）

**预期结果**:
- 强制升级在较短时间内执行
- 升级成功完成
- 正在执行的任务可能被中断（这是强制升级的预期行为）

### 场景4: 升级失败回滚

**前置条件**:
- Agent正常运行，版本1.0.0
- 服务端有新版本1.1.0（假设有问题）
- 当前无任务执行

**测试步骤**:
1. 正常升级流程开始
2. 升级器下载并替换新版本
3. 升级器尝试启动新版本
4. 新版本启动失败（模拟故障）
5. 升级器检测到启动失败
6. 升级器开始回滚流程
7. 升级器恢复原版本文件
8. 升级器启动原版本Agent

**预期结果**:
- 升级失败被检测到
- 自动回滚到原版本
- 原版本Agent恢复正常运行
- 升级失败状态被记录

### 场景5: 升级中的任务分发控制

**前置条件**:
- Agent正在升级过程中
- 服务端有新任务需要分发

**测试步骤**:
1. Agent开始升级流程
2. Agent状态设置为UPGRADING
3. 其他Agent请求任务：正常分发
4. 升级中Agent请求任务：返回空列表
5. 升级完成后Agent状态恢复为ONLINE
6. Agent再次请求任务：正常分发

**预期结果**:
- 升级中的Agent不接收新任务
- 其他Agent正常接收任务
- 升级完成后恢复正常任务分发

## 技术实现要点

### 1. 心跳集成版本检查
```java
// Agent端处理心跳响应
Map<String, Object> heartbeatResponse = api.heartbeat(agentId, agentToken, includeSystemInfo);
handleHeartbeatResponse(heartbeatResponse, upgradeExecutor);
```

### 2. 任务状态监控
```java
// 任务开始时注册
taskStatusMonitor.onTaskStart(executionId);

// 任务完成时注销
taskStatusMonitor.onTaskComplete(executionId);

// 检查是否可以升级
if (upgradeExecutor.canUpgrade()) {
    upgradeExecutor.executeUpgrade(versionInfo);
} else {
    upgradeExecutor.scheduleUpgradeRetry(versionInfo);
}
```

### 3. 独立升级器调用
```java
// 启动升级器
ProcessBuilder pb = new ProcessBuilder();
pb.command("java", "-jar", "upgrader.jar", 
          newVersionPath, agentHome, 
          upgradeLogId.toString(), serverUrl);
pb.start();

// 主程序退出
System.exit(0);
```

### 4. 跨平台兼容性
```java
// Windows启动命令
pb.command("cmd", "/c", "start", "/b", "java", "-jar", "agent.jar");

// Linux/macOS启动命令  
pb.command("nohup", "java", "-jar", "agent.jar", "&");
```
