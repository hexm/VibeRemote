## 📋 问题描述

用户反馈服务器健康度面板存在以下问题：
1. **下方空白太多** - 面板布局不够紧凑，存在大量空白区域
2. **数据不准确** - 内存和磁盘数据显示不准确，缺乏详细信息
3. **指标不够丰富** - 只有6个基础指标，无法全面反映服务器状态

## 🎯 优化方案

### 1. 数据收集优化
- **使用JMX API**: 替换简单的Runtime API，使用ManagementFactory获取准确的系统信息
- **内存信息**: 使用MemoryMXBean获取堆内存使用情况，显示已用/最大内存
- **CPU信息**: 使用OperatingSystemMXBean获取系统负载和CPU核心数
- **磁盘信息**: 获取应用运行目录的磁盘使用情况，显示已用/总容量
- **线程信息**: 获取当前线程数和峰值线程数
- **垃圾回收**: 统计GC次数和总耗时
- **类加载**: 显示已加载的类数量
- **数据库连接**: 获取HikariCP连接池的活跃/总连接数

### 2. 界面布局重设计
- **3行布局**: 改为3行紧凑布局，消除空白区域
- **背景色区分**: 每个指标使用不同的背景色，提高可读性
- **详细信息**: 每个指标显示主要数值和详细信息
- **响应式设计**: 保持在不同屏幕尺寸下的良好显示效果

## 📊 新增监控指标

### 系统资源指标
1. **CPU使用率** - 系统负载百分比 + CPU核心数
2. **内存使用率** - 堆内存使用百分比 + 已用/最大内存(MB)
3. **磁盘使用率** - 磁盘使用百分比 + 已用/总容量(GB)

### JVM运行指标
4. **运行时间** - JVM启动时间(小时+分钟)
5. **活跃线程** - 当前线程数 + 峰值线程数
6. **已载类数** - JVM加载的类数量

### 性能指标
7. **GC统计** - 垃圾回收次数 + 总耗时(ms)
8. **数据库连接** - 活跃连接数/总连接数
