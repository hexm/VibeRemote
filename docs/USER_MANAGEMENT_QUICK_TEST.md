# 用户管理和Agent分组功能 - 快速测试指南

## 测试环境准备

### 1. 启动后端服务
```bash
cd server
mvn spring-boot:run
```

后端将在 http://localhost:8080 启动

### 2. 启动前端服务
```bash
cd web-modern
npm install  # 首次运行需要安装依赖
npm run dev
```

前端将在 http://localhost:3001 启动

### 3. 默认管理员账号
- 用户名：`admin`
- 密码：`admin123`
- 权限：所有权限（16个）

---

## 测试场景

### 场景1：用户管理功能测试

#### 1.1 登录系统
1. 访问 http://localhost:3001
2. 使用默认管理员账号登录
3. 验证登录成功，进入仪表盘

#### 1.2 创建新用户
1. 点击左侧菜单"用户管理"
2. 点击"创建用户"按钮
3. 填写信息：
   - 用户名：`testuser`
   - 密码：`Test1234`（至少8位，包含字母和数字）
   - 邮箱：`test@example.com`
   - 真实姓名：`测试用户`
4. 点击"操作员"快捷模板按钮
5. 验证权限自动勾选（11个权限）
6. 点击"确定"
7. 验证创建成功，用户出现在列表中

#### 1.3 编辑用户权限
1. 找到刚创建的用户
2. 点击"编辑"按钮
3. 修改邮箱为 `test2@example.com`
4. 点击"只读"模板按钮
5. 验证权限变为4个
6. 点击"确定"
7. 验证更新成功，权限数量显示为4

#### 1.4 重置用户密码
1. 点击用户的"重置密码"按钮
2. 输入新密码：`NewPass123`
3. 点击"确定"
4. 验证重置成功

#### 1.5 禁用/启用用户
1. 点击用户的"禁用"按钮
2. 验证状态变为"禁用"（红色标签）
3. 点击"启用"按钮
4. 验证状态变为"启用"（绿色标签）

#### 1.6 删除用户
1. 点击用户的"删除"按钮
2. 在确认对话框中点击"确定"
3. 验证用户从列表中消失

---

### 场景2：Agent分组功能测试

#### 2.1 创建Agent分组
1. 点击左侧菜单"Agent分组"
2. 点击"创建分组"按钮
3. 填写信息：
   - 分组名称：`生产环境`
   - 分组类型：选择"环境分组"
   - 描述：`生产环境服务器`
4. 点击"确定"
5. 验证创建成功，分组出现在列表中

#### 2.2 添加Agent到分组
1. 确保至少有一个Agent在线（启动agent服务）
2. 点击分组的"查看"按钮
3. 在右侧Drawer中，点击"选择要添加的Agent"下拉框
4. 选择一个或多个Agent
5. 点击下拉框外部（触发onBlur）
6. 验证Agent出现在分组成员列表中

#### 2.3 从分组移除Agent
1. 在分组详情中，找到一个Agent
2. 点击"移除"按钮
3. 在确认对话框中点击"确定"
4. 验证Agent从列表中消失

#### 2.4 编辑分组
1. 关闭Drawer
2. 点击分组的"编辑"按钮
3. 修改描述为：`生产环境服务器（已更新）`
4. 点击"确定"
5. 验证更新成功

#### 2.5 创建多个分组
创建以下分组用于后续测试：
- 测试环境（环境分组）
- 开发环境（环境分组）
- 华东区域（地域分组）

---

### 场景3：按分组创建任务测试

#### 3.1 准备工作
1. 确保至少有2个Agent在线
2. 确保有至少一个分组包含多个Agent

#### 3.2 手动选择Agent创建任务
1. 点击左侧菜单"任务管理"
2. 点击"创建任务"按钮
3. 填写信息：
   - 任务名称：`手动选择测试`
   - 选择方式：保持"手动选择"
   - 目标节点：手动选择2个Agent
   - 脚本类型：Shell
   - 脚本内容：`echo "Hello from $(hostname)"`
   - 超时时间：300
   - 自动启动：勾选
4. 点击"创建任务"
5. 验证任务创建成功
6. 查看任务详情，验证有2个执行实例

#### 3.3 按分组选择Agent创建任务
1. 点击"创建任务"按钮
2. 填写信息：
   - 任务名称：`分组选择测试`
   - 选择方式：选择"按分组选择"
   - 选择分组：选择"生产环境"
   - 验证目标节点自动填充
   - 脚本类型：Shell
   - 脚本内容：`echo "Group task from $(hostname)"`
   - 超时时间：300
   - 自动启动：勾选
3. 点击"创建任务"
4. 验证任务创建成功
5. 查看任务详情，验证执行实例数量等于分组中的Agent数量

#### 3.4 分组选择后手动调整
1. 点击"创建任务"按钮
2. 选择方式：选择"按分组选择"
3. 选择分组：选择一个有多个Agent的分组
4. 切换选择方式回"手动选择"
5. 验证Agent列表保持填充状态
6. 手动移除一个Agent
7. 创建任务
8. 验证任务只分发到剩余的Agent

## 常见问题排查

### 问题1：无法登录
- 检查后端服务是否启动（8080端口）
- 检查数据库迁移是否成功
- 检查默认管理员用户是否创建

### 问题2：Agent列表为空
- 检查是否有Agent服务在运行
- 检查Agent是否成功注册到服务器
- 访问 http://localhost:8080/api/web/agents 查看API响应

### 问题3：分组列表为空
- 需要先创建分组
- 检查API是否正常：http://localhost:8080/api/web/agent-groups

### 问题4：权限列表不显示
- 检查后端PermissionConstants是否正确定义
- 检查API是否正常：http://localhost:8080/api/web/permissions

### 问题5：创建任务失败
- 检查是否选择了至少一个Agent
- 检查脚本内容是否为空
- 查看浏览器控制台错误信息
- 查看后端日志

## API测试（可选）

### 使用curl测试API

#### 1. 登录获取token
```bash
curl -X POST http://localhost:8080/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"admin123"}'
```

#### 2. 获取用户列表
```bash
curl -X GET http://localhost:8080/api/web/users \
  -H "Authorization: Bearer YOUR_TOKEN"
```

#### 3. 获取分组列表
```bash
curl -X GET http://localhost:8080/api/web/agent-groups \
  -H "Authorization: Bearer YOUR_TOKEN"
```

#### 4. 获取权限列表
```bash
curl -X GET http://localhost:8080/api/web/permissions \
  -H "Authorization: Bearer YOUR_TOKEN"
```
