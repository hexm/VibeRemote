## 测试步骤

### 测试1：创建草稿任务

1. 打开前端界面：http://localhost:3001
2. 点击"创建任务"按钮
3. 填写任务信息：
   - 任务名称：test-draft-frontend
   - 选择在线节点
   - 脚本类型：Shell
   - 脚本内容：echo "Draft task from frontend"
   - 超时时间：300
4. **关闭"立即启动"开关**
5. 点击"创建任务"
6. 验证：
   - 提示信息："任务创建成功（草稿状态）"
   - 任务列表中出现新任务
   - 任务状态显示为"草稿"（灰色标签）
   - 操作列显示"启动"按钮（绿色）

### 测试2：启动草稿任务

1. 找到刚创建的草稿任务
2. 点击"启动"按钮
3. 在弹出的对话框中选择执行节点
4. 点击"启动"
5. 验证：
   - 提示信息："任务已启动，创建了1个执行实例"
   - 任务状态变为"待执行"（蓝色标签）
   - 操作列显示"停止"按钮（红色）
6. 等待几秒，刷新页面
7. 验证：
   - 任务状态变为"执行中"（蓝色动画标签）
   - 执行进度显示：1/1 已完成
8. 再等待几秒，刷新页面
9. 验证：
   - 任务状态变为"成功"（绿色标签）
   - 操作列显示"重启"按钮（橙色）

### 测试3：创建立即启动的任务

1. 点击"创建任务"按钮
2. 填写任务信息：
   - 任务名称：test-auto-start-frontend
   - 选择在线节点
   - 脚本类型：Shell
   - 脚本内容：echo "Auto start task from frontend"
   - 超时时间：300
3. **保持"立即启动"开关开启**（默认状态）
4. 点击"创建任务"
5. 验证：
   - 提示信息："任务创建成功"
   - 任务列表中出现新任务
   - 任务状态显示为"待执行"（蓝色标签）
   - 操作列显示"停止"按钮

### 测试4：停止运行中的任务

1. 点击"创建任务"按钮
2. 填写任务信息：
   - 任务名称：test-stop-frontend
   - 选择在线节点
   - 脚本类型：Shell
   - 脚本内容：sleep 60
   - 超时时间：300
3. 保持"立即启动"开关开启
4. 点击"创建任务"
5. 等待任务开始执行（状态变为"执行中"）
6. 点击"停止"按钮
7. 在确认对话框中点击"确定"
8. 验证：
   - 提示信息："任务已停止，取消了1个执行实例"
   - 任务状态变为"已取消"（灰色标签）
   - 操作列显示"重启"按钮

### 测试5：状态筛选

1. 在任务列表顶部找到状态筛选下拉框
2. 选择"草稿"
3. 验证：只显示草稿状态的任务
4. 选择"执行中"
5. 验证：只显示执行中的任务
6. 选择"成功"
7. 验证：只显示成功的任务
8. 选择"全部状态"
9. 验证：显示所有任务

### 测试6：查看任务详情

1. 点击任意任务的"查看详情"按钮
2. 验证任务详情模态框显示：
   - 任务状态标签（使用正确的颜色）
   - 目标节点数量
   - 成功/失败/运行中统计
   - 执行进度条
   - 脚本类型、创建者、创建时间
   - 执行实例列表

### 测试7：重启任务

1. 找到一个已完成的任务（成功或失败）
2. 点击"重启"按钮
3. 在弹出的对话框中选择重启模式：
   - 重启所有执行实例
   - 仅重启失败的执行实例
4. 点击"确定重启"
5. 验证：
   - 提示信息："任务已重启，创建了X个新执行实例"
   - 任务状态变为"待执行"
   - 执行次数递增
