## 日志格式标准化

### 1. 日志级别使用
- **INFO**: 关键业务流程和状态变更
- **DEBUG**: 详细的技术信息和调试数据
- **WARN**: 非致命错误和警告信息
- **ERROR**: 严重错误和异常情况

### 2. 日志标识符
- **[TASK-{executionId}]**: 任务相关日志
- **✓**: 成功操作标识
- **✗**: 失败操作标识
- **========**: 重要流程分隔符

### 3. 敏感信息处理
- Agent Token只显示前10个字符
- 脚本内容只显示长度，不显示具体内容
- 系统路径和文件信息适当脱敏

## 日志文件结构

### 1. 统一日志文件 (agent.log)
- 所有Agent主要流程日志
- 启动、注册、心跳、任务分发
- 错误处理和系统状态

### 2. 任务执行日志 (agent-task.log)
- 任务执行详细过程
- 脚本输出和错误信息
- 任务状态变更

### 3. 升级日志 (agent-upgrade.log)
- Agent升级相关操作
- 版本检查和下载过程
- 升级状态和结果

## 配置建议

### 1. 生产环境
```properties
log.level=INFO
log.file.max.size=50MB
log.file.max.count=10
```

### 2. 开发环境
```properties
log.level=DEBUG
log.file.max.size=10MB
log.file.max.count=5
```

### 3. 故障排查
```properties
log.level=DEBUG
# 临时启用详细日志进行问题诊断
```
