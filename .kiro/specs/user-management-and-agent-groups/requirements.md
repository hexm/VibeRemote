# 用户管理和客户端分组功能 - 需求文档

## 功能概述

本需求文档描述了LightScript系统的两个核心功能：
1. **用户管理**：支持创建用户、分配权限，实现细粒度的访问控制
2. **客户端分组**：支持将Agent按业务、环境等维度进行分组管理

## 1. 用户管理功能

### 1.1 功能背景

当前系统使用简单的用户认证机制，缺乏完善的用户管理和权限控制功能。需要实现：
- 管理员可以创建、编辑、删除用户
- 用户直接绑定权限（不使用角色层，保持简单）
- 不同用户具有不同的操作权限
- 操作审计和日志记录

### 1.2 用户故事

#### 1.2.1 管理员创建用户
**作为** 系统管理员  
**我想要** 创建新用户并分配权限  
**以便** 让团队成员使用系统并控制他们的权限

**验收标准**：
- 管理员可以访问用户管理页面
- 可以创建新用户，输入用户名、密码、邮箱、真实姓名
- 可以为用户直接分配权限（通过勾选权限列表）
- 用户名必须唯一
- 密码需要满足安全要求（至少8位，包含字母和数字）
- 创建成功后显示在用户列表中

#### 1.2.2 管理员编辑用户
**作为** 系统管理员  
**我想要** 编辑现有用户的信息和权限  
**以便** 更新用户权限或修正错误信息

**验收标准**：
- 可以编辑用户的邮箱、真实姓名、权限
- 可以重置用户密码
- 不能修改用户名
- 不能编辑自己的权限（防止误操作）
- 修改后立即生效

#### 1.2.3 管理员删除/禁用用户
**作为** 系统管理员  
**我想要** 删除或禁用用户账号  
**以便** 撤销离职员工或不再需要的账号的访问权限

**验收标准**：
- 可以禁用用户（软删除，保留数据）
- 可以永久删除用户（硬删除，删除所有数据）
- 禁用的用户无法登录
- 不能删除自己的账号
- 删除前需要确认操作

#### 1.2.4 权限管理
**作为** 系统管理员  
**我想要** 为用户分配权限  
**以便** 灵活控制用户的操作范围

**验收标准**：
- 系统定义固定的权限列表
- 创建/编辑用户时可以直接勾选权限
- 权限包括：
  - 用户管理（创建、编辑、删除、查看用户）
  - 任务管理（创建、执行、删除、查看任务）
  - 脚本管理（创建、编辑、删除、查看脚本）
  - Agent管理（查看、分组）
  - 日志查看
  - 系统设置

#### 1.2.5 用户查看自己的权限
**作为** 普通用户  
**我想要** 查看自己拥有的权限  
**以便** 了解我可以执行哪些操作

**验收标准**：
- 用户可以在个人中心查看自己的权限列表
- 界面上只显示用户有权限的功能菜单
- 无权限的操作按钮显示为禁用状态或不显示

### 1.3 权限列表

| 权限代码 | 权限名称 | 描述 |
|---------|---------|------|
| user:create | 创建用户 | 可以创建新用户 |
| user:edit | 编辑用户 | 可以编辑用户信息 |
| user:delete | 删除用户 | 可以删除用户 |
| user:view | 查看用户 | 可以查看用户列表 |
| role:manage | 角色管理 | 可以管理角色和权限 |
| task:create | 创建任务 | 可以创建新任务 |
| task:execute | 执行任务 | 可以启动/停止任务 |
| task:delete | 删除任务 | 可以删除任务 |
| task:view | 查看任务 | 可以查看任务列表和详情 |
| script:create | 创建脚本 | 可以创建新脚本 |
| script:edit | 编辑脚本 | 可以编辑脚本 |
| script:delete | 删除脚本 | 可以删除脚本 |
| script:view | 查看脚本 | 可以查看脚本列表 |
| agent:view | 查看Agent | 可以查看Agent列表 |
| agent:group | Agent分组 | 可以管理Agent分组 |
| log:view | 查看日志 | 可以查看执行日志 |
| system:settings | 系统设置 | 可以修改系统设置 |

### 1.4 权限模板（前端快捷功能）

为方便快速分配权限，前端提供快捷模板按钮：

#### 1.4.1 管理员模板
- 一键勾选所有权限
- 适用于系统管理员

#### 1.4.2 操作员模板
- 任务管理：创建、执行、删除、查看
- 脚本管理：创建、编辑、删除、查看
- Agent管理：查看、分组
- 日志查看
- 适用于日常运维人员

#### 1.4.3 只读模板
- 只勾选查看类权限（task:view, script:view, agent:view, log:view）
- 适用于只需要查看的用户

### 1.5 数据模型

#### User（用户表）
```
- id: Long (主键)
- username: String (用户名，唯一)
- password: String (密码，BCrypt加密存储)
- email: String (邮箱)
- realName: String (真实姓名)
- status: String (状态：ACTIVE, DISABLED)
- createdAt: LocalDateTime (创建时间)
- updatedAt: LocalDateTime (更新时间)
- lastLoginAt: LocalDateTime (最后登录时间)
```

#### UserPermission（用户权限关联表）
```
- id: Long (主键)
- userId: Long (用户ID，外键)
- permissionCode: String (权限代码)
```

**权限代码说明**：
- 直接使用字符串存储权限代码（如 "task:create"）
- 不需要单独的权限表，权限列表在代码中定义
- 简化数据库设计，提高查询性能

## 2. 客户端分组功能

### 2.1 功能背景

当前系统的Agent管理是扁平化的，当Agent数量增多时，管理和操作变得困难。需要实现分组功能：
- 按业务、环境、地域等维度对Agent进行分组
- 支持批量操作分组内的Agent
- 提升Agent管理效率

### 2.2 用户故事

#### 2.2.1 创建Agent分组
**作为** 系统管理员或操作员  
**我想要** 创建Agent分组  
**以便** 将Agent按业务或环境进行组织

**验收标准**：
- 可以创建新分组，输入分组名称和描述
- 分组名称必须唯一
- 可以选择分组类型（业务、环境、地域等）
- 创建成功后显示在分组列表中

#### 2.2.2 将Agent加入分组
**作为** 系统管理员或操作员  
**我想要** 将Agent加入到分组中  
**以便** 对Agent进行分类管理

**验收标准**：
- 可以在分组详情页面添加Agent
- 可以从Agent列表中选择多个Agent批量加入分组
- 一个Agent可以属于多个分组
- 加入后立即在分组中显示

#### 2.2.3 从分组中移除Agent
**作为** 系统管理员或操作员  
**我想要** 从分组中移除Agent  
**以便** 调整分组成员

**验收标准**：
- 可以在分组详情页面移除Agent
- 可以批量移除多个Agent
- 移除操作不影响Agent本身，只是解除关联关系

#### 2.2.4 查看分组信息
**作为** 用户  
**我想要** 查看分组列表和详情  
**以便** 了解Agent的组织结构

**验收标准**：
- 可以查看所有分组列表
- 显示分组名称、描述、Agent数量
- 可以点击分组查看详情
- 分组详情显示所有成员Agent

#### 2.2.5 按分组创建任务
**作为** 操作员  
**我想要** 为整个分组创建任务  
**以便** 批量执行脚本

**验收标准**：
- 创建任务时可以选择分组
- 选择分组后自动选中该分组的所有Agent
- 可以在分组基础上再手动调整Agent选择
- 任务执行时针对分组内的所有Agent

#### 2.2.6 编辑和删除分组
**作为** 系统管理员或操作员  
**我想要** 编辑或删除分组  
**以便** 维护分组信息

**验收标准**：
- 可以编辑分组名称和描述
- 可以删除空分组
- 删除非空分组时需要确认
- 删除分组不影响Agent本身

### 2.3 分组类型

| 类型代码 | 类型名称 | 描述 |
|---------|---------|------|
| BUSINESS | 业务分组 | 按业务系统分组 |
| ENVIRONMENT | 环境分组 | 按开发、测试、生产环境分组 |
| REGION | 地域分组 | 按地理位置分组 |
| CUSTOM | 自定义分组 | 用户自定义的分组 |

### 2.4 数据模型

#### AgentGroup（Agent分组表）
```
- id: Long (主键)
- name: String (分组名称，唯一)
- description: String (描述)
- type: String (分组类型：BUSINESS, ENVIRONMENT, REGION, CUSTOM)
- createdBy: String (创建者)
- createdAt: LocalDateTime (创建时间)
- updatedAt: LocalDateTime (更新时间)
```

#### AgentGroupMember（分组成员关联表）
```
- id: Long (主键)
- groupId: Long (分组ID)
- agentId: String (Agent ID)
- addedAt: LocalDateTime (加入时间)
```

## 3. 功能优先级

### 3.1 第一阶段（核心功能）
1. ✅ 用户管理基础功能（创建、编辑、删除用户）
2. ✅ 角色和权限管理
3. ✅ 权限验证和访问控制
4. ✅ Agent分组基础功能（创建、编辑、删除分组）
5. ✅ Agent加入/移除分组

### 3.2 第二阶段（增强功能）
1. 按分组创建任务
2. 分组统计和监控
3. 操作审计日志
4. 用户活动记录

### 3.3 第三阶段（高级功能）
1. 分组权限控制（用户只能操作特定分组）
2. 分组层级结构（父子分组）
3. 分组模板
4. 批量导入导出

## 4. 非功能性需求

### 4.1 安全性
- 密码必须加密存储（BCrypt）
- 敏感操作需要二次确认
- 登录失败次数限制
- 会话超时自动登出
- 操作审计日志

### 4.2 性能
- 用户列表分页加载
- 权限验证缓存
- 分组查询优化
- 支持1000+用户
- 支持100+分组

### 4.3 可用性
- 界面简洁直观
- 操作流程清晰
- 错误提示友好
- 支持批量操作
- 支持搜索和筛选

### 4.4 兼容性
- 向后兼容现有用户数据
- 平滑升级，不影响现有功能
- 支持数据迁移

### 5.1 用户管理验收标准
