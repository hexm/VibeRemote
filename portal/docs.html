LightScript 文档

现代化的分布式脚本执行平台

什么是 LightScript？

LightScript 是一个现代化的分布式脚本执行平台，专为运维团队和开发者设计。它提供了强大的多节点管理、智能任务调度和实时监控功能，让您的脚本执行更加高效、可靠和可视化。

核心特性

  • 分布式执行：支持多节点并行执行，自动负载均衡
  • 实时监控：可视化监控面板，实时查看执行状态和性能指标
  • 智能调度：基于资源使用情况的智能任务调度算法
  • 多语言支持：支持 Shell、Python、PowerShell 等多种脚本语言
  • 安全可靠：完善的权限管理、审计日志和安全执行环境
  • 易于部署：支持 Docker 容器化部署，简单快捷

架构概览

LightScript 采用客户端-服务端架构：

  • 服务端：提供 Web 管理界面、API 接口和任务调度功能
  • 客户端：部署在目标服务器上，负责接收和执行脚本任务
  • Web 界面：现代化的管理后台，提供完整的管理功能

安装指南

快速部署 LightScript 服务端和客户端

服务端安装

Docker 部署（推荐）

# 下载 docker-compose.yml
curl -O https://raw.githubusercontent.com/lightscript/lightscript/main/docker-compose.yml

# 启动服务
docker-compose up -d

手动部署

# 下载服务端
wget https://github.com/lightscript/lightscript/releases/latest/download/lightscript-server.jar

# 启动服务端
java -jar lightscript-server.jar

客户端安装

Linux/macOS

# 一键安装脚本
curl -fsSL http://your-server/scripts/install-linux.sh | bash -s -- --server=http://your-server:8080

Windows

# PowerShell 安装
Set-ExecutionPolicy Bypass -Scope Process -Force
iex ((New-Object System.Net.WebClient).DownloadString('http://your-server/scripts/install-windows.ps1'))

快速开始

5分钟内创建并执行您的第一个脚本任务

步骤 1：登录管理后台

打开浏览器访问 http://your-server:3000，使用默认账号登录：

  • 用户名：admin
  • 密码：admin123

步骤 2：添加客户端

在客户端管理页面，您可以看到已连接的客户端节点。如果没有客户端，请先安装客户端。

步骤 3：创建脚本

在脚本管理页面，点击"新建脚本"，输入脚本内容：

#!/bin/bash
echo "Hello, LightScript!"
echo "当前时间: $(date)"
echo "系统信息: $(uname -a)"

步骤 4：执行任务

在任务管理页面，点击"创建任务"，选择脚本和目标客户端，然后点击"执行"。

步骤 5：查看结果

在任务详情页面，您可以实时查看任务执行状态、输出日志和执行结果。

客户端管理

管理和监控所有连接的客户端节点

客户端状态

客户端有以下几种状态：

  • 在线：客户端正常连接，可以接收任务
  • 离线：客户端断开连接或停止运行
  • 忙碌：客户端正在执行任务

客户端分组

您可以将客户端按照环境、用途等维度进行分组管理：

  • 创建分组：在分组管理页面创建新的分组
  • 添加客户端：将客户端加入到相应的分组
  • 批量操作：对整个分组执行批量任务

资源监控

实时监控客户端的资源使用情况：

  • CPU 使用率
  • 内存使用率
  • 磁盘使用率
  • 网络连接状态

脚本管理

创建、编辑和管理您的脚本库

支持的脚本类型

  • Shell：Linux/macOS 系统脚本
  • PowerShell：Windows 系统脚本
  • Python：跨平台 Python 脚本
  • Batch：Windows 批处理脚本

脚本编辑器

内置的代码编辑器提供以下功能：

  • 语法高亮
  • 代码补全
  • 错误检查
  • 代码格式化

脚本参数

您可以为脚本定义参数，在执行时动态传入：

#!/bin/bash
# 脚本参数示例
SERVER_NAME=$1
BACKUP_PATH=$2

echo "备份服务器: $SERVER_NAME"
echo "备份路径: $BACKUP_PATH"

任务执行

创建、执行和监控脚本任务

创建任务

创建任务时需要指定：

  • 任务名称
  • 执行脚本
  • 目标客户端或分组
  • 脚本参数（可选）
  • 超时时间

任务状态

  • 等待中：任务已创建，等待执行
  • 运行中：任务正在执行
  • 成功：任务执行成功
  • 失败：任务执行失败
  • 超时：任务执行超时
  • 取消：任务被手动取消

实时日志

任务执行过程中，您可以实时查看：

  • 标准输出
  • 错误输出
  • 执行进度
  • 资源使用情况

监控面板

实时监控系统状态和性能指标

仪表盘概览

仪表盘提供系统整体状态的快速概览：

  • 在线客户端数量
  • 任务执行统计
  • 系统资源使用情况
  • 最近任务执行趋势

性能监控

实时监控关键性能指标：

  • 任务执行成功率
  • 平均执行时间
  • 客户端负载分布
  • 错误率统计

告警通知

支持多种告警方式：

  • 邮件通知
  • 短信通知
  • Webhook 集成
  • 钉钉/企业微信通知

API 认证

了解如何使用 LightScript API

获取 API Token

在管理后台的"个人设置"页面生成 API Token。

认证方式

在请求头中添加 Authorization 字段：

Authorization: Bearer YOUR_API_TOKEN

API 基础 URL

http://your-server:8080/api/v1

客户端 API

管理客户端的 API 接口

获取客户端列表

GET /api/v1/agents

curl -H "Authorization: Bearer YOUR_TOKEN" \
     http://your-server:8080/api/v1/agents

获取客户端详情

GET /api/v1/agents/{agentId}

curl -H "Authorization: Bearer YOUR_TOKEN" \
     http://your-server:8080/api/v1/agents/agent-001

删除客户端

DELETE /api/v1/agents/{agentId}

curl -X DELETE \
     -H "Authorization: Bearer YOUR_TOKEN" \
     http://your-server:8080/api/v1/agents/agent-001

任务 API

任务管理的 API 接口

创建任务

POST /api/v1/tasks

curl -X POST \
     -H "Authorization: Bearer YOUR_TOKEN" \
     -H "Content-Type: application/json" \
     -d '{
       "name": "系统检查",
       "scriptContent": "#!/bin/bash\necho \"Hello World\"",
       "scriptLang": "shell",
       "agentIds": ["agent-001"],
       "timeoutSec": 300
     }' \
     http://your-server:8080/api/v1/tasks

获取任务列表

GET /api/v1/tasks

curl -H "Authorization: Bearer YOUR_TOKEN" \
     http://your-server:8080/api/v1/tasks

获取任务详情

GET /api/v1/tasks/{taskId}

curl -H "Authorization: Bearer YOUR_TOKEN" \
     http://your-server:8080/api/v1/tasks/task-001
