Linux 服务端手工部署

适用于公司内网、不可直连发布环境的场景。先从门户下载服务端部署包，再上传到目标 Linux 服务器解压， 修改一份配置文件并执行部署脚本，即可完成门户、控制台前端、后端和 Agent 安装资源的统一部署。

1. 下载服务端部署包

推荐先从当前门户下载最新版部署包，再传到目标服务器进行离线部署。页面会优先显示最新构建信息；如果清单不可用，也可以直接使用固定下载地址。

部署包下载入口为固定地址：/downloads/viberemote-server-deploy-package-latest.tar.gz。如门户已同步构建清单，页面会自动显示最新版本信息。

服务端部署脚本执行完成后，也会在目标环境门户中同步发布当前部署包和清单，便于后续复用。

2. 环境准备

  • 一台 Linux 服务器，预装 Java 8
  • 同一台服务器预装 Nginx
  • 一个可访问的 MySQL 数据库
  • 数据库只需要先建库，不需要手工建表

3. 数据库初始化说明

服务端首次启动时会自动创建表结构和基础数据。
  • 业务表结构：自动创建
  • 默认管理员：admin / admin123
  • 系统参数：自动初始化
  • 前提：数据库账号需要有建表和改表权限

4. 手工部署步骤

  1. 1

    上传并解压部署包

    先从门户下载部署包，再上传到目标 Linux 服务器。以下示例假定部署包已位于当前目录。

    tar -xzf viberemote-server-deploy-package-latest.tar.gz
cd viberemote-server-deploy-package-*
  2. 2

    复制并修改部署配置

    部署包里自带 deploy.env.example。你只需要复制为 deploy.env 并填写目标环境信息。

    cp deploy.env.example deploy.env
vi deploy.env
  3. 3

    执行服务端部署脚本

    脚本会自动检测 Java、Nginx、数据库连通性、部署用户、部署包完整性，并完成门户、前端、后端与 Agent 安装资源发布。

    bash install-server.sh
  4. 4

    执行部署后验证

    如需单独再次验证，可以重复执行验证脚本。

    bash verify-deploy.sh

5. deploy.env 关键配置项

以下字段是首次部署时最需要确认的配置。其余字段部署包已提供默认值，可按需调整。

PORTAL_PUBLIC_BASE_URL 门户公网或内网访问地址，例如 http://10.10.10.20
ADMIN_PUBLIC_BASE_URL 控制台前端地址，例如 http://10.10.10.20:3000
API_PUBLIC_BASE_URL 后端地址，例如 http://10.10.10.20:8080
DB_URL MySQL JDBC 地址，库需要你提前建好
DB_USERNAME / DB_PASSWORD 数据库账号与密码，建议具备建表和改表权限
REGISTER_TOKEN Agent 注册令牌，一键安装脚本会自动写入客户端配置
WEB_ENCRYPTION_KEY 控制台前后端通信加密密钥，建议使用随机生成值
AGENT_PACKAGE_BASE_URL 留空时，会按 PORTAL_PUBLIC_BASE_URL/agent/release 自动推导。
JWT_SECRET 留空时，会自动生成并保存到部署目录下的 deploy-generated.env。
本页只关注服务端首次部署。Agent 自动升级属于后续运维维护工作，不在首次部署步骤内。

6. 部署脚本会自动完成什么

  • 检测 Java 8、Nginx、数据库连通性
  • 生成 application-prod.yml
  • 部署后端 server.jar
  • 部署控制台前端到 Nginx 管理端口
  • 部署门户静态站点
  • 发布 Agent 安装包与一键安装脚本
  • 生成 Nginx 配置和后端启停脚本
  • 创建备份、启动服务并执行健康检查

7. 部署后如何验证

bash /opt/viberemote/bin/status-backend.sh
nginx -t

curl http://127.0.0.1:8080/actuator/health
curl http://127.0.0.1:3000/
curl http://127.0.0.1/agent/release/version.json

首次登录控制台后，请尽快修改默认管理员密码。
