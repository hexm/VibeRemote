# LightScript 分布式脚本执行管理系统 - 需求分析文档

## 📋 项目概述

### 项目背景
LightScript 是一个功能架构与 XZB-1248/Spark 类似的分布式脚本执行管理系统，旨在为企业提供统一的跨平台脚本执行和管理能力。系统采用服务器-客户端架构，支持批量脚本下发、实时监控和集中管理。

### 项目目标
- 构建企业级分布式脚本执行管理平台
- 实现跨平台客户端统一管理
- 提供安全可靠的批量脚本执行能力
- 建立完善的监控和日志体系
- 保持轻量级设计，最小化资源占用

## 🎯 核心需求分析

### 1. 服务器端需求

#### 1.1 Web管理界面
**功能需求：**
- ✅ **用户登录认证**
  - 支持用户名/密码登录
  - JWT令牌认证机制
  - 会话管理和权限控制
  - 默认管理员账户：admin/admin123

- ✅ **客户端列表查看**
  - 实时显示在线/离线客户端状态
  - 客户端基本信息展示（主机名、IP、操作系统、CPU、内存等）
  - 客户端连接时间和最后心跳时间
  - 支持客户端搜索和筛选

- ✅ **客户端基本信息管理**
  - 查看客户端详细系统信息
  - 客户端状态监控
  - 支持客户端备注和标签管理
  - 客户端分组管理功能

**技术需求：**
- 前端使用Vue.js 3框架
- 响应式设计，支持多设备访问
- Element Plus UI组件库
- 前后端分离架构

#### 1.2 状态检测机制
**功能需求：**
- ✅ **心跳检测**
  - 客户端定期发送心跳包（默认30秒间隔）
  - 服务器端监控客户端在线状态
  - 超时自动标记为离线状态
  - 支持心跳间隔配置

- ✅ **状态同步**
  - 实时更新客户端状态信息
  - 系统资源使用情况监控
  - 客户端版本信息同步
  - 异常状态告警机制

#### 1.3 脚本管理与执行
**功能需求：**
- ✅ **脚本下发**
  - 支持向单个客户端下发脚本
  - 支持批量脚本下发（一对多）
  - 脚本类型支持：Bash、PowerShell、CMD
  - 脚本参数传递和环境变量设置

- ✅ **执行监控**
  - 实时查看脚本执行状态
  - 执行进度跟踪
  - 执行结果统计和分析
  - 支持任务取消和重试

- ✅ **日志管理**
  - 完整的执行日志记录
  - 实时日志流显示
  - 日志搜索和过滤功能
  - 日志导出和归档

### 2. 客户端Agent需求

#### 2.1 跨平台支持
**功能需求：**
- ✅ **Windows系统支持**
  - 支持Windows 7/8/10/11/Server版本
  - CMD和PowerShell脚本执行
  - Windows服务形式运行（可选）
  - 系统资源监控

- ✅ **Linux系统支持**
  - 支持主流Linux发行版（Ubuntu、CentOS、RHEL等）
  - Bash脚本执行支持
  - 守护进程运行模式
  - 系统信息采集

#### 2.2 资源优化
**技术需求：**
- ✅ **轻量级设计**
  - Java 1.8运行时，最小化依赖
  - 内存占用控制在50MB以内
  - CPU使用率保持在5%以下（空闲状态）
  - 网络带宽占用最小化
  - 如果Java占用资源过多，考虑其他技术方案

- ✅ **性能优化**
  - 异步任务执行机制
  - 连接池和资源复用
  - 优雅的启动和关闭流程
  - 内存泄漏防护

#### 2.3 通信机制
**功能需求：**
- ✅ **自动注册**
  - 启动时自动向服务器注册
  - 注册令牌验证机制
  - 重连机制和故障恢复
  - 网络异常处理

- ✅ **任务执行**
  - 定期拉取待执行任务
  - 安全的脚本执行环境
  - 执行结果实时上报
  - 异常情况处理和上报

## 🏗️ 技术架构需求

### 3.1 服务器端技术栈
**已确定技术选型：**
- ✅ **后端框架**：Spring Boot 2.7.18 + Java 1.8
- ✅ **数据库**：MySQL 8.0（无外部缓存）
- ✅ **安全框架**：Spring Security + JWT认证
- ✅ **数据访问**：Spring Data JPA + Hibernate
- ✅ **前端框架**：Vue.js 3 + Element Plus
- ✅ **构建工具**：Maven 3.6+

### 3.2 客户端技术栈
**技术选型：**
- ✅ **主要语言**：Java 1.8（与服务器端保持一致）
- ✅ **HTTP客户端**：Java 11 HttpClient（向后兼容）
- ✅ **JSON处理**：Jackson
- ✅ **日志框架**：Logback
- ✅ **打包方式**：Maven Assembly Plugin（独立JAR）

### 3.3 部署架构
**部署需求：**
- ✅ **前后端分离**：Web界面独立部署
- ✅ **数据库独立**：MySQL单独部署
- ✅ **Agent分布式**：支持多台机器部署
- ✅ **配置管理**：环境变量和配置文件管理

**重要约束：**
- ❗ **必须使用bat命令**，不使用PowerShell
- ✅ 当前项目已提供完整的bat启动脚本

```bash

build.bat

# 服务器启动
start-server.bat

# 客户端启动

# Web前端启动

# 快速启动
```
