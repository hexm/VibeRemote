# 升级后启动指南

## 快速启动

### 1. 确认配置文件
检查 `server/src/main/resources/application.yml`，确认日志配置：
```yaml
lightscript:
  log:
    storage:
      path: logs/tasks  # 确保路径正确
```

### 2. 数据库自动升级
首次启动时，系统会自动执行数据库迁移：
- 增加 `execution_count` 字段
- 增加 `log_file_path` 字段
- 更新已有任务的执行次数

**无需手动操作！**

### 3. 启动服务器
```bash
cd server
mvn spring-boot:run
```

### 4. 验证升级
1. 访问 Web 界面：http://localhost:8080
2. 创建并执行一个测试任务
3. 查看任务日志 - 应该能正常显示
4. 点击"下载日志" - 应该能下载日志文件
5. 检查日志文件是否生成：`logs/tasks/2024/XX/`

### 5. 确认日志文件
检查日志文件是否按预期生成：
```
logs/
  tasks/
    2024/
      01/
        {taskId}_1_{timestamp}.log  ← 应该能看到新的日志文件
```

## 常见问题

### Q: 启动失败，提示数据库迁移错误
**A**: 检查数据库是否已经有 `execution_count` 字段，如果有，可能是重复迁移。
```sql
-- 检查字段是否存在
DESCRIBE tasks;
```

### Q: 日志文件没有生成
**A**: 检查以下几点：
1. 配置文件中的路径是否正确
2. 服务器是否有写入权限
3. 任务是否已启动（status=RUNNING）
4. 检查服务器日志是否有错误信息

### Q: 前端日志显示为空
**A**: 
1. 打开浏览器开发者工具，查看API请求
2. 确认 `/web/tasks/{taskId}/logs` 接口返回数据
3. 检查任务的 `logFilePath` 字段是否有值

### Q: 下载日志按钮无响应
**A**: 
1. 检查日志文件是否存在
2. 确认 `/web/tasks/{taskId}/logs/download` 接口能访问
3. 查看浏览器控制台是否有错误

## 测试步骤

### 完整功能测试
1. **创建任务**
   - 创建一个简单的测试任务
   - 确认任务创建成功

2. **Agent拉取任务**
   - Agent应该能正常拉取任务
   - 任务状态变为 PULLED

3. **执行任务**
   - Agent ACK任务，状态变为 RUNNING
   - 检查 `execution_count` 是否变为 1
   - 检查 `log_file_path` 是否生成

4. **查看日志**
   - 在Web界面点击"日志"按钮
   - 应该能看到实时日志
   - 显示总行数

5. **下载日志**
   - 点击"下载日志"按钮
   - 应该能下载完整的日志文件
   - 文件名格式正确

6. **任务完成**
   - 任务执行完成
   - 状态变为 SUCCESS/FAILED
   - 日志文件保留

## 性能对比

### 升级前（数据库存储）
- 1万行日志 → 1万次数据库INSERT
- 查询日志需要JOIN操作
- 大日志可能导致数据库膨胀

### 升级后（文件存储）
- 1万行日志 → 1万次文件追加（更快）
- 查询日志直接读文件
- 日志大小不受限制

## 监控建议

### 1. 磁盘空间
定期检查日志目录大小：
```bash
du -sh logs/tasks/
```

### 2. 日志文件数量
统计日志文件数量：
```bash
find logs/tasks/ -name "*.log" | wc -l
```

### 3. 最大日志文件
查找最大的日志文件：
```bash
find logs/tasks/ -name "*.log" -exec ls -lh {} \; | sort -k5 -hr | head -10
```

## 回滚方案

如果升级后遇到问题需要回滚：

1. 停止服务器
2. 恢复代码到升级前版本
3. 执行数据库回滚（参考《日志存储升级说明.md》）
4. 重启服务器
