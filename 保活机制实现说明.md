# LightScript Agent 保活与自恢复机制实现说明

## 📋 问题描述

**原问题**: 服务器重启后，agent的心跳请求会被拒绝并返回400错误，导致agent无法继续工作。

**根本原因**: 
1. 服务器重启后，agent使用的旧token在新的服务器实例中无效
2. 服务端在agent重新注册时，虽然复用了记录但没有生成新token
3. Agent端没有自动重新注册的机制

## ✅ 解决方案

### 1. 服务端修改

**文件**: `server/src/main/java/com/example/lightscript/server/service/AgentService.java`

#### 修改点1: Agent重新注册时生成新token

```java
if (existingAgent.isPresent()) {
    agent = existingAgent.get();
    agent.setIp(request.getIp());
    agent.setLabels(request.getLabels());
    agent.setLastHeartbeat(LocalDateTime.now());
    agent.setStatus("ONLINE");
    // ✅ 新增：生成新的token，确保服务器重启后agent可以重新建立连接
    agent.setAgentToken(UUID.randomUUID().toString());
    agent = agentRepository.save(agent);
}
```

**作用**: 每次agent重新注册时都会获得一个新的token，解决服务器重启后token失效的问题。

#### 修改点2: 心跳失败时增加详细日志

```java
@Transactional
public boolean updateHeartbeat(String agentId, String agentToken, HeartbeatRequest request) {
    Optional<Agent> agentOpt = agentRepository.findByAgentIdAndAgentToken(agentId, agentToken);
    if (agentOpt.isPresent()) {
        // 心跳成功逻辑...
        return true;
    }
    
    // ✅ 新增：Token验证失败时的详细日志
    Optional<Agent> agentById = agentRepository.findById(agentId);
    if (agentById.isPresent()) {
        log.warn("[Agent] Heartbeat failed - Token mismatch for agent {} ({}). Agent needs to re-register.", 
                agentById.get().getHostname(), agentId);
    } else {
        log.warn("[Agent] Heartbeat failed - Unknown agent ID: {}. Agent needs to register.", agentId);
    }
    
    return false;
}
```

**作用**: 提供更清晰的日志信息，帮助诊断问题。

### 2. Agent端修改

**文件**: `agent/src/main/java/com/example/lightscript/agent/AgentMain.java`

#### 修改点1: 启动时带重试的注册机制

```java
// 注册Agent（带重试机制）
String agentId = null;
String agentToken = null;
int retryCount = 0;
int retryDelay = 1000; // 初始延迟1秒

while (agentId == null) {
    try {
        System.out.println("Registering agent" + (retryCount > 0 ? " (attempt " + (retryCount + 1) + ")..." : "..."));
        Map<String, Object> reg = api.register(registerToken, hostname, osType);
        agentId = String.valueOf(reg.get("agentId"));
        agentToken = String.valueOf(reg.get("agentToken"));
        
        System.out.println("Agent registered successfully!");
    } catch (Exception e) {
        retryCount++;
        System.err.println("Failed to register agent: " + e.getMessage());
        System.out.println("Retrying in " + (retryDelay / 1000) + " seconds...");
        
        Thread.sleep(retryDelay);
        
        // 指数退避：1s -> 2s -> 5s -> 10s -> 30s (max)
        if (retryDelay < 2000) {
            retryDelay = 2000;
        } else if (retryDelay < 5000) {
            retryDelay = 5000;
        } else if (retryDelay < 10000) {
            retryDelay = 10000;
        } else {
            retryDelay = 30000; // 最大30秒
        }
    }
}
```

**作用**: Agent启动时如果服务器不可达，会自动重试直到连接成功或用户退出。

#### 修改点2: 运行时自动重新注册机制

```java
// 连接状态追踪
final boolean[] needReRegister = {false};
final String[] currentAgentId = {finalAgentId};
final String[] currentAgentToken = {finalAgentToken};

// 主循环中检查是否需要重新注册
if (needReRegister[0]) {
    System.out.println("========================================");
    System.out.println("Connection lost. Re-registering agent...");
    System.out.println("========================================");
    
    // 执行重新注册逻辑（带指数退避）
    while (!reRegistered && !Thread.currentThread().isInterrupted()) {
        try {
            Map<String, Object> reg = api.register(registerToken, hostname, osType);
            currentAgentId[0] = String.valueOf(reg.get("agentId"));
            currentAgentToken[0] = String.valueOf(reg.get("agentToken"));
            
            // 更新任务执行器的凭证
            taskRunner.updateCredentials(currentAgentId[0], currentAgentToken[0]);
            
            System.out.println("Agent re-registered successfully!");
            needReRegister[0] = false;
            reRegistered = true;
        } catch (Exception e) {
            // 重试逻辑...
        }
    }
}
```

**作用**: 运行时检测到连接问题后自动重新注册，无需手动重启agent。

#### 修改点3: 心跳失败检测与触发重连

```java
int heartbeatFailures = 0;
final int MAX_HEARTBEAT_FAILURES = 3; // 连续失败3次触发重连

// 心跳检测 - 每30秒一次
if (now - lastHeartbeat > 30_000) {
    try {
        api.heartbeat(currentAgentId[0], currentAgentToken[0]);
        heartbeatFailures = 0; // 重置失败计数
    } catch (Exception e) {
        heartbeatFailures++;
        System.err.println("Heartbeat failed (" + heartbeatFailures + "/" + MAX_HEARTBEAT_FAILURES + "): " + e.getMessage());
        
        if (heartbeatFailures >= MAX_HEARTBEAT_FAILURES) {
            System.err.println("Max heartbeat failures reached. Triggering re-registration...");
            needReRegister[0] = true;
            heartbeatFailures = 0;
        }
    }
}
```

**作用**: 监控心跳状态，连续失败3次后自动触发重新注册。

#### 修改点4: 任务操作失败时的认证错误检测

```java
try {
    api.ack(currentAgentId[0], currentAgentToken[0], taskId);
} catch (Exception e) {
    // Token可能失效，标记需要重新注册
    if (e.getMessage().contains("400") || e.getMessage().contains("401") || e.getMessage().contains("403")) {
        System.err.println("Authentication error detected. Will re-register...");
        needReRegister[0] = true;
    }
}
```

**作用**: 在任务确认、日志上报等操作中检测认证错误，及时触发重连。

### 3. SimpleTaskRunner修改

**文件**: `agent/src/main/java/com/example/lightscript/agent/SimpleTaskRunner.java`

#### 修改点: 支持凭证动态更新

```java
class SimpleTaskRunner {
    private final AgentApi api;
    private volatile String agentId;      // ✅ 改为volatile可变
    private volatile String agentToken;   // ✅ 改为volatile可变
    private volatile boolean shutdown = false;
    
    /**
     * 更新agent凭证（在重新注册后调用）
     */
    synchronized void updateCredentials(String newAgentId, String newAgentToken) {
        this.agentId = newAgentId;
        this.agentToken = newAgentToken;
        System.out.println("[TaskRunner] Credentials updated");
    }
}
```

**作用**: 允许在重新注册后更新任务执行器使用的凭证，确保正在执行的任务也能正常上报结果。

## 🔄 工作流程

### 场景1: Agent启动时服务器不可达

```
1. Agent启动
2. 尝试注册 -> 失败
3. 等待1秒后重试
4. 尝试注册 -> 失败
5. 等待2秒后重试
6. ...（指数退避，最大30秒）
7. 直到服务器恢复或用户退出
```

### 场景2: 运行中服务器重启

```
1. Agent正常运行
2. 服务器重启
3. Agent发送心跳 -> 失败（400错误）
4. 记录失败次数 = 1
5. Agent发送心跳 -> 失败（400错误）
6. 记录失败次数 = 2
7. Agent发送心跳 -> 失败（400错误）
8. 记录失败次数 = 3，达到阈值
9. 触发重新注册流程
10. 尝试重新注册 -> 成功
11. 获得新的agentId和agentToken
12. 更新所有组件的凭证
13. 恢复正常工作（心跳、任务拉取等）
```

### 场景3: 网络临时中断

```
1. Agent正常运行
2. 网络中断
3. 心跳失败 -> 记录失败次数 = 1
4. 心跳失败 -> 记录失败次数 = 2
5. 网络恢复
6. 心跳成功 -> 重置失败次数 = 0
7. 继续正常工作（未触发重新注册）
```

## 📝 配置参数

| 参数 | 默认值 | 说明 |

| 心跳间隔 | 30秒 | 两次心跳之间的间隔 |

## ✨ 关键特性

### 1. **启动时连接重试**
- Agent启动时如果无法连接服务器，会持续重试
- 使用指数退避策略避免频繁重试
- 仅当用户手动退出时才停止重连

### 2. **运行时自动恢复**
- 检测到网络问题或服务器不可达时自动进入恢复模式
- 保持现有任务状态，暂停新任务拉取
- 连接恢复后自动重新注册
- 全程无需手动重启Agent

### 3. **健康检查机制**
- 心跳作为主要健康检查手段
- 连续失败达到阈值后触发重连
- 任务操作失败时也会进行连接检查

### 4. **资源占用控制**
- 使用指数退避策略避免CPU和网络资源浪费
- 重连机制不会显著增加内存占用
- 所有重试操作都可被用户中断

## 🌐 服务端任务管理

除了Agent端的保活机制，服务端提供简洁的任务管理机制。详见《服务端恢复支持说明.md》。

### 服务端关键机制

#### 1. Agent重连时的处理
```java
// AgentService.java  
public RegisterResponse register(RegisterRequest request) {
    // 生成新token，确保agent能重新建立连接
    agent.setAgentToken(UUID.randomUUID().toString());
    agent.setStatus("ONLINE");
    
    // ⚠️ 不重置任务！
    // Agent重连不代表任务失败，任务可能还在执行中
}
```

#### 2. 任务超时机制
```java
@Scheduled(fixedRate = 300000) // 每5分钟
public void checkTimeoutTasks() {
    // 标记超过30分钟的RUNNING任务为TIMEOUT
    // 不自动重置任务，由人工决定是否重新执行
}
```

**为什么不自动重置失败的任务？**
```
问题场景：
- Agent网络短暂中断，任务还在执行
- 如果自动重置任务 → 任务被执行两次！

某些任务不应该被重复执行：
- 支付任务
- 数据删除任务  
- 一次性配置任务
- 发送通知任务

正确做法：
- 任务超时后标记为TIMEOUT
- 由人工在Web界面查看并决定是否重新执行
```

#### 3. 任务保障机制

**保障1**: PULLED任务超时回退（2分钟未ACK → PENDING）
**保障2**: RUNNING任务超时标记（30分钟 → TIMEOUT）
**保障3**: Agent状态管理（2分钟无心跳 → OFFLINE）

## 总结

通过Agent端和服务端的协同配合，LightScript系统现在具备了完整的保活和自恢复能力：

### Agent端能力
✅ Agent启动时如果服务器连接不上，会持续重试直到用户退出
✅ 运行过程中遇到网络问题或服务器重启，环境恢复后自动恢复通信
✅ 使用指数退避策略优化资源占用
✅ 提供详细的日志信息便于问题诊断
✅ 无需手动重启agent即可恢复正常工作

### 服务端能力
✅ Agent重新注册时生成新token，确保认证有效
✅ **不自动重置任务**：避免任务重复执行的风险
✅ **超时机制**：RUNNING任务30分钟超时标记为TIMEOUT
✅ **人工决策**：由用户判断是否重新执行超时/失败的任务
✅ 完整的任务状态转换和日志记录

### 端到端保障
✅ **任务不重复执行**: 不自动重置任务，避免重复执行风险
✅ **状态明确**: TIMEOUT/FAILED/SUCCESS状态清晰
✅ **连接自动恢复**: Agent和服务端协同实现无缝重连
✅ **人工可控**: 重新执行任务由人工判断
✅ **安全可靠**: 避免某些不应该被重复执行的任务被自动执行（如支付、删除等）

### 关键设计决策
⚠️ **不自动重置失败的任务** - 由人工判断是否重新执行
✅ **依赖超时机制** - 标记超时任务，不自动重置
✅ **业务安全** - 避免支付、删除等任务被错误重复执行

这使得LightScript系统成为一个真正安全、可控、可靠的分布式脚本执行管理平台，可以应对各种异常场景。
